🏛️ ClassSync — Complete Setup Guide

Step-by-step instructions for setting up the Classroom Management System from absolute zero. No prior experience needed.
Phase 1

⬇️ Download & Install XAMPP

1 What is XAMPP?

XAMPP is a free program that puts a web server (Apache) and a database (MySQL) on your laptop. Think of it like a mini internet server that lives only on your computer. You need it to run PHP files.

2 Download XAMPP

Go to this website in your browser:
https://www.apachefriends.org/download.html

Click the big Download button for Windows. The file is about 160 MB. Wait for it to finish downloading.

3 Install XAMPP

1. Double-click the downloaded file (it looks like xampp-windows-x64-8.x.x-installer.exe).

2. If Windows asks "Do you want to allow this app to make changes?" click Yes.

3. Click Next through all the screens. Leave everything at the default settings.

4. The install location will be C:\xampp — do NOT change this.

5. Click Finish when done.

💡 If your antivirus warns you, it's a false alarm. XAMPP is safe and used by millions of developers worldwide.
4 Start XAMPP

1. Search for XAMPP Control Panel in the Windows Start menu and open it.

2. You will see a panel with rows: Apache, MySQL, FileZilla, etc.

3. Click the Start button next to Apache. The row should turn green.

4. Click the Start button next to MySQL. That row should also turn green.

✅ Both Apache and MySQL should show green. That means your local server is running!
⚠️ If Apache fails to start, another program (like Skype or IIS) might be using port 80. In XAMPP Control Panel → Config → httpd.conf, change "Listen 80" to "Listen 8080". Then use localhost:8080 instead of localhost everywhere in this guide.
Phase 2

🗄️ Set Up the Database

5 Open phpMyAdmin

phpMyAdmin is a visual interface to manage your MySQL database — like a spreadsheet app but for databases.

Open your browser and go to:

http://localhost/phpmyadmin

You should see a blue/white page with database tools. No login is needed by default.

6 Import the Database Schema

1. In phpMyAdmin, click Import in the top menu bar.

2. Click Choose File.

3. Navigate to wherever you saved the project files and select classroom_db.sql.

4. Scroll down and click the Import button (it's blue, at the bottom right).

5. Wait a few seconds. You should see a green success message.

✅ You should now see "classroom_mgmt" appear in the left sidebar of phpMyAdmin. Clicking it will show you all the tables: departments, faculty, classrooms, etc.
💡 The SQL file automatically creates the database, all tables, sample data, triggers, stored procedures, and views. You don't need to do anything else in the database.
7 Verify the Data Loaded

In phpMyAdmin's left sidebar, click classroom_mgmt, then click faculty. You should see 7 teachers listed including Mrs. Jyoti Sharma.

Click classrooms — you should see 8 rooms including B811.

Phase 3

📁 Set Up the Project Files

8 Create the Project Folder

Your XAMPP server serves files from a special folder called htdocs. Any file placed here can be accessed in a browser.

Open Windows Explorer (the file manager) and navigate to:

C:\xampp\htdocs

Create a new folder here called classroom.

Your full path should be: C:\xampp\htdocs\classroom

9 Copy the Project Files

Copy these 3 files you downloaded into the classroom folder:

C:\xampp\htdocs\classroom\ ├── index.html ← The main website (connected to MySQL) ├── api.php ← The PHP backend (talks to MySQL) └── classroom_db.sql ← Already imported, just keep it here
💡 Make sure the files are directly inside the classroom folder, not inside another subfolder.
10 Check api.php Settings

Open api.php with Notepad (right-click → Open with → Notepad).

Near the top, look for these lines:

$host = "localhost";
$user = "root";
$pass = "";        // default XAMPP password is empty
$db   = "classroom_mgmt";

For a fresh XAMPP install, these settings are already correct — the MySQL username is root and the password is blank. You don't need to change anything.

⚠️ If you previously set a password for MySQL, enter it in the $pass line like: $pass = "yourpassword";
Phase 4

🚀 Run the Application

11 Open the Application

Make sure Apache and MySQL are still running (green) in the XAMPP Control Panel.

Open your browser (Chrome is recommended) and go to:

http://localhost/classroom/index.html

The ClassSync dashboard should load with a navy blue interface.

✅ In the top-right corner you should see: ⬤ MySQL Connected — this means the website is talking to your database successfully!
⚠️ If you see "⬤ DB Offline" in red, double-check that MySQL is running in XAMPP and that the api.php database settings are correct.
12 Test the Check-In Feature

Let's simulate Mrs. Jyoti entering room B811 to teach Processor Architecture to SY IT students:

1. Click Check-In in the left sidebar.

2. Select Mrs. Jyoti Sharma from the Faculty dropdown.

3. Select B811 (Classroom) from the Room dropdown.

4. Select Processor Architecture (IT201) from the Subject dropdown.

5. Select Year 2 Div-A · Information Technology from the Class dropdown.

6. Click the blue ✅ Check In button.

7. Now click Dashboard — you will see B811 is now marked Occupied with Mrs. Jyoti's session live!

✅ This is stored directly in your MySQL database. Go to phpMyAdmin → classroom_mgmt → live_sessions to see the record!
Phase 5

📘 How the System Works

13 File Roles Explained
FileWhat it does
index.htmlThe visual website. Everything you see and click. Uses JavaScript to fetch data from api.php and display it. No page reloads.
api.phpThe middleman. Receives requests from index.html, runs MySQL queries, and sends back the data as JSON.
classroom_db.sqlThe database blueprint. Creates all tables, sample data, triggers, stored procedures, and views. Only needed once during import.
14 The Flow — What Happens When You Click Check-In
You click "Check In" in the browser
       ↓
index.html sends a request to api.php?action=checkin
       ↓
api.php calls MySQL stored procedure: sp_faculty_checkin()
       ↓
MySQL TRIGGER fires: checks if room is already occupied
       ↓
If free → inserts into live_sessions table
Another TRIGGER fires → writes to audit_log automatically
       ↓
api.php sends back "SUCCESS"
       ↓
index.html shows green toast: "Check-in successful!"
Dashboard updates to show the room as Occupied
15 Database Tables Overview
TableStores
departmentsIT, CS, ELEX, MECH departments
facultyAll teachers — name, code, email, designation
classroomsRooms — B811, LAB101 etc. with capacity & amenities
subjectsCourses like Processor Architecture, DBMS, etc.
yearsClasses — FY IT-A, SY CS-B, etc.
timetable_slotsPre-planned weekly schedule (Mon–Sat)
live_sessionsReal-time check-ins (who is teaching where right now)
reservationsAdvance room bookings
audit_logEvery action ever taken — auto-filled by triggers
Phase 6

🔧 Troubleshooting Common Problems

16 "DB Offline" shown in top-right

Problem: The website can't connect to MySQL.

Fix checklist:

1. Is MySQL running (green) in XAMPP Control Panel? If not, click Start next to MySQL.

2. Did the SQL import succeed? Check phpMyAdmin for the classroom_mgmt database.

3. Did you open the page as http://localhost/classroom/index.html? (Not by double-clicking the file!)

17 "Apache not starting" error

Problem: Port 80 is already used by another program.

Fix: In XAMPP Control Panel, click Config next to Apache → httpd.conf. Find the line Listen 80 and change it to Listen 8080. Save and restart Apache. Then open the app at http://localhost:8080/classroom/index.html. Also update the first line in index.html where it says const API = "http://localhost/classroom/api.php" to http://localhost:8080/classroom/api.php.

18 Page loads but dropdowns are empty

Problem: api.php can't be found or has an error.

Fix: Open http://localhost/classroom/api.php?action=get_faculty in your browser. You should see JSON text. If you see a blank page or PHP error, check that api.php is in the right folder.

19 "Room already in use" even for empty rooms

Problem: A previous test session is still marked "ongoing" in the database.

Fix: Go to phpMyAdmin → classroom_mgmt → live_sessions → find the row where status = 'ongoing' → click Edit → change status to completed → click Save. Or click Check-Out in the app.
